# RevA12 Menu Selection Screen

## Purpose

The RevA12 menu layer gives KoalaByte Blue a single selection screen that can be driven by:

- Six front-panel GPIO buttons.
- Touchscreen drag scrolling.
- Touchscreen long-press select.
- Keyboard input during development.

## Default menu

```text
eucalyptus      Always-on passive BLE scanner/logger
Koala Kapture   Capture and archive BLE advertisement metadata
Koala Kry       Replay captured metadata into the UI/report/XP pipeline
Ear Tag         Named lab BLE beacon
Urban Poaching  Authorized BLE RSSI lab game
Reports         Generate and review reports
Settings        Device and companion settings
Lab             Password-gated authorized lab menu
Shutdown        Confirm safe shutdown
```

## Button mapping

The menu uses the existing six-button GPIO mapping:

```text
Button 1 = Main Menu
Button 2 = Left / Back
Button 3 = Select; hold for Shutdown
Button 4 = Right / Forward
Button 5 = Up
Button 6 = Down
```

Button events are generated by:

```text
pi-companion/koalablue/gpio_buttons.py
```

Menu state is handled by:

```text
pi-companion/koalablue/menu_ui.py
```

## Touchscreen behavior

Touchscreen controls are display-backend agnostic:

```text
on_touch_down(y)
on_touch_move(y)
on_touch_up(y)
```

Supported behavior:

```text
Drag up/down       Scroll menu
Tap row            Focus row
Long press row     Select row
```

Default long-press threshold:

```text
0.75 seconds
```

## Test from terminal

```bash
cd pi-companion
source .venv/bin/activate
PYTHONPATH=. python ../scripts/run_menu_screen.py
```

Keyboard test controls:

```text
w = up
s = down
a = left/back
d = right/forward
Enter = select
m = main menu
q = quit
```

## Event log

Menu events are logged locally:

```text
logs/menu_events.jsonl
```

## Safety scope

The menu only routes local UI actions. Sensitive lab features should continue to use the password-gated authorized lab menu and existing safe-mode boundaries.
